Konteks:
Teks berikut adalah hasil OCR dari tulisan tangan mahasiswa
berupa pseudocode / kode program sederhana.

Tujuan:
Memperbaiki kesalahan OCR dengan PERUBAHAN MINIMAL
agar teks lebih terbaca, TANPA mengubah struktur asli OCR,
dan tetap kompatibel dengan evaluasi CER.

Aturan STRUKTUR (WAJIB):
- JANGAN menambah baris baru.
- JANGAN menghapus baris apa pun.
- JANGAN mengubah urutan baris.
- JANGAN menambahkan teks tambahan di luar isi OCR.
- Jumlah baris output HARUS SAMA dengan input OCR.
- Jangan menambahkan keterangan, komentar, atau label baru.

Aturan PERBAIKAN (BOLEH):
1. Memperbaiki typo huruf pada keyword atau variabel
   (contoh: "prograe" → "program", "inreger" → "integer").
2. Memperbaiki spasi yang salah atau berlebih.
3. Memperbaiki simbol yang JELAS salah OCR dengan perubahan minimal.

Aturan KHUSUS domain pseudocode:
- Semua kata/token pada output HARUS berasal dari teks OCR.
  DILARANG menambahkan kata baru yang tidak ada di OCR.
- Pada perintah input(...), pemisah variabel adalah koma (,), bukan titik (.).
- Tanda titik (.) hanya BOLEH digunakan untuk akses struct
  (contoh: S.nama, data.umur). Jika tidak ada indikasi struct,
  titik pada input dianggap salah OCR dan diganti koma (,).
- Nama variabel TIDAK BOLEH berupa simbol matematika (+, -, *, /).
  Jika simbol tersebut muncul sebagai variabel akibat OCR,
  perbaiki menjadi huruf alfabet yang paling mirip secara visual
  (contoh: '+' → 't'), tanpa menebak logika baru.
- Jangan mengubah ekspresi matematika atau rumus,
  kecuali koreksi simbol yang jelas salah baca OCR.

Larangan KERAS:
- Jangan merekonstruksi algoritma.
- Jangan menambah atau menghapus statement.
- Jangan menafsirkan ulang maksud program.
- Jangan memperbaiki baris yang tidak jelas maknanya.

Output:
Tampilkan HANYA teks hasil perbaikan.
Tanpa penjelasan.
Tanpa markdown.
Satu baris output untuk satu baris input.

Input OCR:
<<<
{OCR_TEXT}
>>>
